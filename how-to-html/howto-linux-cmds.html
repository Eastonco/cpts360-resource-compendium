How do I download a file from the website?

The sample code and/or executables may be downloaded from the course website, specifically at the following URL: https://eecs.wsu.edu/~cs360/samples/

You may notice the webpage looks something like this:

Let’s say you want to download the a.out executable and out text file from the LAB1pre/ directory. Let’s click on LAB1pre/ and see its contents:

There are a few ways we can download these files. The first way is to use the curl command:

curl <URL> --output <filename>

For this, we must include the name of the file we’re downloading in the URL we provide curl. For example, if we want to download a.out, it would look like this:

curl https://eecs.wsu.edu/~cs360/samples/LAB1pre/a.out --output a.out

Similarly, to download out,

curl https://eecs.wsu.edu/~cs360/samples/LAB1pre/out --output out

A perhaps simpler way is to use wget (you can run sudo apt-get install wget if it is not already installed):

wget https://eecs.wsu.edu/~cs360/samples/LAB1pre/a.out

How do I run a program/executable?

In order to run a program, you need to type ./ before the name of the executable. Therefore, to run a.out, you must type:

./a.out

However, without further action upon downloading the executable, you may notice the following error:

This is because you do not have permission to run this program by default. To give yourself permission, you must use the chmod command:

chmod +x <filename>

The +x flag gives all user groups in Linux permission to execute a given executable. You can see this in the permission bits of the file (on the left side) before and after running chmod +x:

Now running the command via ./a.out should work.

How do I use Git/GitHub Classroom?

On the Canvas or course webpage, click the “accept” link for each relevant assignment. You should see a screen like this:

Click Accept this assignment. This will clone a template repository for that assignment. After a second, you should be able to refresh and see a link to your repository. Simply click the link.

Your repository by default should look something like the following:

At this point, you will want to clone (download) the repository so you may work on it on your own computer. Since we are using Git for version control, you will use Git commands to accomplish this.

To download the template repository, use the git clone command in the terminal:

git clone <URL>

where the URL is your repository URL. In my case, it would be the following:

git clone https://github.com/cpts-360-spring22/github-test-assignment-zjnett

(Tip: you can use CTRL+Shift+V to paste text into a Linux terminal.)

Also note this command may have you sign in to your GitHub account. If you have not already set up a personal access token (or SSH key), you will need to do so (see here for more info).

From here, you now have a copy of the repository on your local computer. You can create files in this directory or edit files in your favorite editor.

To add your changes to the repository, use the command git add:

git add .

This will add all changed files to the staging index of Git— this essentially preps files to be committed, or “saved” to the repository.

To summarize all changed files and “save” them to the repository, use git commit:

git commit -m "Your message here"

Note the string above will be your own commit message— you can use something like “Initial commit” or “Add main.c”. Best practice is to have your commit message summarize the changes you made.

Note you may be asked to run git config and specify your email and name: please do so using your GitHub email address and your real name. An example using my information would be something like:

git config --global user.email "zachary.nett@wsu.edu"
git config --global user.name "Zach Nett"

How do I use XYZ Linux command?

Luckily, there’s a built-in command to learn about other commands called man:

man <command name>

Therefore, if you want to learn about the ls command, you can do the following:

man ls

Which will pop up with a window that looks like this:

Note this is displayed using the less command— you can use arrow keys (or page up/page down) to scroll and q to quit.

To search a man page, you can pipe the command with the grep tool (we’ll learn about pipes later in this course):

man <command name> | grep "<search string here>"

For example, if I want to sort the ls man page for information on sorting, I can do:

man ls | grep "sort"
